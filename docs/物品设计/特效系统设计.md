# 物品特效系统设计

> 更新时间：2026-03-05

---

## 一、onTrigger 字段规范

物品触发时执行的侧效果，定义在各物品 tiers 数据中：

```json
"onTrigger": {
  "adjacentCharge": 2000,
  "damageGrowth":   5,
  "healGrowth":     3,
  "shieldGrowth":   3,
  "burnGrowth":     3,
  "poisonGrowth":   2,
  "multiTrigger":   3
}
```

### 7 种效果说明

| 字段 | 效果 | 备注 |
|---|---|---|
| `adjacentCharge` | 触发时为左右相邻物品各充能 N ms | 无相邻物品时无效 |
| `damageGrowth` | 每次触发后本物品 damage +N（本战） | 战斗结束重置 |
| `healGrowth` | 每次触发后本物品 heal +N（本战） | 战斗结束重置 |
| `shieldGrowth` | 每次触发后本物品 shield +N（本战） | 战斗结束重置 |
| `burnGrowth` | 每次触发后本物品 burn +N（本战） | 战斗结束重置 |
| `poisonGrowth` | 每次触发后本物品 poison +N（本战） | 战斗结束重置 |
| `multiTrigger` | 基础效果与所有 onTrigger 侧效果重复执行 N 次 | 身份技能不受影响 |

多种效果可同时存在于同一物品。

## 二、特效池（config/specialEffects.json）

独立于物品数据之外的随机特效池，供事件系统（深锚强化、破釜沉舟）动态赋予物品特效使用。

```json
[
  { "id": "damageGrowth",   "label": "伤害递增", "values": { "Bronze": 4, "Silver": 6,    "Gold": 9,    "Diamond": 14   } },
  { "id": "healGrowth",     "label": "治疗递增", "values": { "Bronze": 1, "Silver": 2,    "Gold": 3,    "Diamond": 5    } },
  { "id": "shieldGrowth",   "label": "护盾递增", "values": { "Bronze": 2, "Silver": 4,    "Gold": 6,    "Diamond": 9    } },
  { "id": "burnGrowth",     "label": "灼烧递增", "values": { "Bronze": 1, "Silver": 2,    "Gold": 3,    "Diamond": 5    } },
  { "id": "poisonGrowth",   "label": "剧毒递增", "values": { "Bronze": 1, "Silver": 2,    "Gold": 3,    "Diamond": 5    } },
  { "id": "multiTrigger",   "label": "双重触发", "values": { "Bronze": 2, "Silver": 2,    "Gold": 2,    "Diamond": 2    } },
  { "id": "adjacentCharge", "label": "相邻充能", "values": { "Bronze": 1000, "Silver": 1500, "Gold": 2000, "Diamond": 3000 } }
]
```

### applyRandomEffect(inst)

定义于 `src/data/specialEffects.js`，对物品实例执行随机特效赋予：

1. 从特效池中随机抽取一种特效
2. 根据 `inst.tier` 查找对应数值
3. 写入 `inst.onTrigger[effectId] = val`（叠加，不覆盖已有特效）
4. 拼接 `inst.skill_cn`（以 `；` 分隔）

## 三、DoT 机制（灼烧 / 剧毒）

每秒触发一次，行为有意不对称：

| DoT 类型 | 护盾交互 | 说明 |
|---|---|---|
| 灼烧（burn） | 先吸收护盾，再扣 HP | 护盾日志记录 `burnAbsorbed` |
| 剧毒（poison） | **穿透护盾，直接扣 HP** | 有意设计：毒是内伤，无视防御 |

衰减：每 tick 后 `stacks = floor(stacks × 0.6)`

视觉浮字：仅显示实际扣 HP 值（灼烧已扣护盾后的剩余量；剧毒即全量）。

## 四、运行时实现

### startBattle 初始化
```js
item._cooldown        = item.cooldown
item._bonusDamage     = 0
item._bonusHeal       = 0
item._bonusShield     = 0
item._bonusBurn       = 0
item._bonusPoison     = 0
```

### triggerItem 执行顺序
1. 合并 bonus：`damage + _bonusDamage`，`heal + _bonusHeal`，`shield + _bonusShield`，`burn + _bonusBurn`，`poison + _bonusPoison`
2. 循环 `multiTrigger` 次（默认1次）：
   - `resolveEffects` 结算基础效果
   - `applyOnTrigger` 执行侧效果
3. 中级技能钩子（`blade_echo`、`poison_spread`）额外结算一次
4. `accumStats` 统计战报（每次 activate 仅统计一次）

### applyOnTrigger 处理逻辑
```js
adjacentCharge → 为 index±1 物品推进 cooldownProgress
damageGrowth   → item._bonusDamage  += val
healGrowth     → item._bonusHeal    += val
shieldGrowth   → item._bonusShield  += val
burnGrowth     → item._bonusBurn    += val
poisonGrowth   → item._bonusPoison  += val
```
