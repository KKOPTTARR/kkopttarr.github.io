# 物品特效系统设计

> 更新时间：2026-03-04

---

## onTrigger 字段规范

```json
"onTrigger": {
  "adjacentCharge": 2000,
  "damageGrowth":   5,
  "healGrowth":     3,
  "shieldGrowth":   3,
  "burnGrowth":     3,
  "poisonGrowth":   2,
  "multiTrigger":   3
}
```

### 7种效果说明

| 字段 | 效果 | 备注 |
|---|---|---|
| `adjacentCharge` | 触发时为左右相邻物品各充能 N ms | 无相邻物品时无效 |
| `damageGrowth` | 每次触发后本物品 damage +N（本战） | 战斗结束重置 |
| `healGrowth` | 每次触发后本物品 heal +N（本战） | 战斗结束重置 |
| `shieldGrowth` | 每次触发后本物品 shield +N（本战） | 战斗结束重置 |
| `burnGrowth` | 每次触发后本物品 burn +N（本战） | 战斗结束重置 |
| `poisonGrowth` | 每次触发后本物品 poison +N（本战） | 战斗结束重置 |
| `multiTrigger` | 基础效果与所有 onTrigger 侧效果重复执行 N 次 | 身份技能不受影响 |

多种效果可同时存在于同一物品。

## 运行时实现

### startBattle 初始化
```js
item._cooldown        = item.cooldown
item._bonusDamage     = 0
item._bonusBurn       = 0
item._bonusPoison     = 0
item._pendingSelfCharge = 0
```

### triggerItem 执行顺序
1. 合并 bonus：`damage + _bonusDamage`，`burn + _bonusBurn`，`poison + _bonusPoison`
2. 循环 `multiTrigger` 次（默认1次）：
   - `resolveEffects` 结算基础效果
   - `applyOnTrigger` 执行侧效果
3. `accumStats` 统计战报
4. 身份技能（`gun_charge` 等）每次 activate 只执行一次

### applyOnTrigger 处理逻辑
```js
adjacentCharge → 为 index±1 物品推进 cooldownProgress
damageGrowth   → item._bonusDamage  += val
healGrowth     → item._bonusHeal    += val
shieldGrowth   → item._bonusShield  += val
burnGrowth     → item._bonusBurn    += val
poisonGrowth   → item._bonusPoison  += val
burnGrowth        → item._bonusBurn   += val
poisonGrowth      → item._bonusPoison += val
```
