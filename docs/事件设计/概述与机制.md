# 事件系统 — 概述与机制

> 更新时间：2026-03-05（代码核查修订）

---

## 一、系统概述

事件数据存储于 `config/events.json`，共 12 个事件模板。
每轮战斗结束后触发 **两轮**事件选择（`EVENTS_PER_ROUND = 2`），玩家从 3 个选项中选 1 个执行。
事件分为**战后事件**（第一轮）和**战前事件**（第二轮），两个阶段的事件池不同。

---

## 二、字段规范

### 事件模板结构

```json
{
  "id":             "upgrade",
  "icon":           "⬆️",
  "img":            "/events/upgrade.png",
  "label":          "指定升级",
  "desc":           "升级一件指定类型的物品",
  "pool":           "normal",
  "condition":      "hasUpgradeable",
  "targetPhase":    "UPGRADE",
  "immediateAction": null
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | 唯一标识符 |
| `pool` | string | 权重池：`normal`（常规）/ `rare`（稀有）/ `last`（最后一事件专用） |
| `condition` | string | 可选，出现条件检查函数名 |
| `targetPhase` | string | 可选，选择后跳转的界面阶段 |
| `immediateAction` | string | 可选，选择后直接执行的动作（无需额外界面） |
| `guaranteed` | bool | 仅 `last` 池使用，保证出现 |

---

## 三、事件流程

### 每轮两次事件

| 顺序 | 阶段 | pendingEvents | 触发时机 |
|---|---|---|---|
| 第一次 | 战后 | 2 | 战斗结束后立即弹出 |
| 第二次 | 战前 | 1 | 玩家整理阵容后、开战前弹出 |

### 事件池分类

| 事件池 | 事件 ID |
|---|---|
| 战后（post） | `wreck_search` / `storm_shuffle` / `gambler_dice` / `tavern_gamble` / `shop` / `break_boats` / `blind_trade` |
| 战前（pre） | `upgrade` / `alchemy` / `exchange` / `deep_anchor` / `combat_shield` |

### 选项数量

| 事件类型 | 普通选项数 |
|---|---|
| 战后（第一次） | 3 |
| 战前（第二次，最后事件）| 2 + 1 guaranteed（临时护盾） |

---

## 四、概率机制

### 稀有事件

稀有事件（`pool: "rare"`）**不额外增加选项**，而是以 **25%** 概率替换最后一个普通选项。

```
RARE_EVENT_CHANCE = 0.25
```

当前稀有事件：`upgrade`（指定升级）、`alchemy`（炼金）

### 章节权重

各章节会通过 `getChapterEventWeights(battleCount, eventPhase)` 对特定事件增加权重（默认权重 ×3），影响该事件被抽到的概率。详见 `config/chapters.json`。

### 出现条件（condition）

| condition 值 | 检查逻辑 |
|---|---|
| `hasItems` / `hasUpgradeable` | 存在可操作的物品 |
| `hasThreeMatching` | 存在 3 件同品质非钻石物品（炼金前提） |
| `hasMultipleItems` | 物品总数 ≥ 2 |

条件不满足的事件将从本轮候选中移除，不会出现在选项里。

---

## 五、触发方式

| 触发类型 | 字段 | 行为 |
|---|---|---|
| 跳转界面 | `targetPhase` | 进入对应操作界面，玩家完成后返回 ARRANGE |
| 立即执行 | `immediateAction` | 直接执行逻辑，无需额外界面 |

当前立即执行事件：

| 事件 | action | 效果 |
|---|---|---|
| 风暴洗牌 | `stormShuffle` | 所有物品随机替换为同品质其他物品 |
| 深锚强化 | `deepAnchor` | 从全部物品中随机选一件，赋予随机永久特效 |
| 临时护盾 | `combatShield` | 标记 `stormBlessingActive`，战斗开始时附加 `50 × 章节数` 点护盾（章节 1=50，章节 2=100，以此类推） |

---

## 六、互斥组机制

同组事件在同一轮中最多出现一个，避免玩家面对同质化选项。互斥组定义于 `useEventSystem.js`：

| 互斥组 | 成员 | 原因 |
|---|---|---|
| 获得物品类 | 残骸搜寻 / 宝藏商店 / 蒙面交易 | 三者均为"免费获取新物品" |
| 掷骰赌博类 | 赌徒的骰子 / 酒馆赌注 | 均为掷骰随机事件，体验重复 |
| 品质提升类 | 指定升级 / 炼金 | 均为提升物品品质，只是代价不同 |
